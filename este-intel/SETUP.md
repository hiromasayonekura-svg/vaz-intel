# エステティック通信 Intel Platform セットアップ手順

VAZ Intel Platform と同じ仕組みのエステ業界版です。所要時間は約20分。

## 全体像

```
Googleニュース → [GAS] 収集 → スプレッドシート → [GAS] Gemini分析 → [GAS] GitHubへ公開 → ダッシュボード
```

- **収集**: 毎朝5時台に監視対象46件(35社+11キーワード)のニュースを自動取得
- **分析**: 毎時、Gemini(無料枠)が6項目スコア+営業/編集/新規事業の3視点で分析
- **公開**: 毎日8時台・17時台に GitHub Pages のダッシュボードへ反映

スコア設計(30点満点):

| 視点 | 項目(各0〜5点) | 使う人 |
|---|---|---|
| 営業 | 出稿ポテンシャル / 成長・投資活発度 | 広告営業(攻め先の判断) |
| 編集 | 読者関心度 / 話題性・新規性 | 編集部(記事ネタの判断) |
| 事業 | 市場インパクト / 転用可能性 | リーダー陣(新規事業の種) |

---

## STEP 1: GitHub側 (設定済み ✅)

エステ版は既存の `vaz-intel` リポジトリに同居する構成で、デプロイ済みです。

- ダッシュボードURL: `https://hiromasayonekura-svg.github.io/vaz-intel/este/`
- データの置き場所: リポジトリ内 `docs/este/data.json`
- GAS用トークン: vaz-intel用に発行済みのPersonal Access Tokenがそのまま使えます

## STEP 2: スプレッドシート作成 (5分)

1. 新規Googleスプレッドシートを作成(名前例:「エステ通信Intel_DB」)
2. シートを4枚作り、名前を正確に以下へ変更:
   - `1_収集ニュース一覧`
   - `2_定量評価スコア`
   - `3_経営分析ダッシュボード`
   - `4_監視対象`
3. `4_監視対象` シートに `監視対象リスト.csv` の内容を貼り付け
   (ファイル > インポート > アップロード > 「現在のシートに追加」)
   - 1〜3のシートはヘッダー含め空でOK(GASが自動生成します)

※監視対象は後からいつでも行の追加・削除・D列のTRUE/FALSE切替で変更できます。

## STEP 3: GAS設定 (7分)

1. スプレッドシートの 拡張機能 > Apps Script を開く
2. `gas/` フォルダの4ファイルの中身を、それぞれ新規スクリプトファイルとして貼り付け:
   - `00_main.gs` / `01_collectNews.gs` / `02_analyzeNews.gs` / `03_exportToGitHub.gs`
3. Gemini APIキーを取得: https://aistudio.google.com/apikey で「APIキーを作成」(無料)
4. Apps Script の ⚙プロジェクトの設定 > スクリプトプロパティ に5つ登録:

| プロパティ | 値 |
|---|---|
| GEMINI_API_KEY | STEP3-3で取得したキー |
| GITHUB_TOKEN | 発行済みのPersonal Access Token(repoスコープ) |
| GITHUB_OWNER | hiromasayonekura-svg |
| GITHUB_REPO | vaz-intel |
| GITHUB_BRANCH | main |

## STEP 4: 動作確認と自動化 (3分)

1. スプレッドシートを再読み込み → メニューに「📡 Intel運用」が出る
2. 「1️⃣ ニュース収集を実行」(初回は権限承認が必要。「詳細」→「安全でないページに移動」で承認)
   → `1_収集ニュース一覧` にニュースが入ればOK
3. 「2️⃣ AI分析を実行」→ シート2・3に分析が入ればOK(1回12件ずつ処理)
4. 「3️⃣ GitHubへ公開」→ ダッシュボードが実データに置き換わればOK
5. 最後に「⏰ 自動実行トリガーを設定」を1回クリック → 以後は毎日全自動

---

## 運用のコツ

- **未処理が溜まっている場合**: 分析は毎時12件ずつ自動処理されるので放置でOK。急ぐなら「2️⃣」を手動で数回実行
- **ノイズが多い企業**: `4_監視対象` のC列の検索キーワードを具体的にする(例: 「ポーラ」→「ポーラ エステ」)
- **足切り調整**: 分析対象の最低スコアは `00_main.gs` の `MIN_SCORE_FOR_ANALYSIS`(初期値8)で調整
- **Gemini無料枠**: モデルは `gemini-2.5-flash`。制限に当たる場合は `GEMINI_SLEEP_MS` を増やすか `ANALYZE_BATCH` を減らす
- **閲覧制限したい場合**: GitHub Pages(Public)は URL を知っていれば誰でも見られます。社外秘情報を扱う場合はリポジトリをPrivateにして GitHub Pages を有料プランで制限するか、Cloudflare Access 等の導入を検討

## ダッシュボードの見方(部署別)

- **営業** タブ: 攻め先ランキング(営業スコア累積)+ 出稿提案の切り口つきニュース
- **編集** タブ: いま読者に刺さるキーワード + 企画の切り口つき記事ネタ候補
- **経営アジェンダ** タブ: 新規事業への示唆 + リーダー陣への問いかけ
- **ホーム**: 全部署共通の俯瞰(カテゴリ別件数・トレンド・部署別TOP打ち手)
